# MP115Cloud - MoviePilot 115 云下载接管插件

这个插件用于 MoviePilot V2：当 MoviePilot 订阅命中资源并准备添加到下载器前，插件会先请求你配置的非公开搜索页；如果页面里有匹配的 `magnet:` 链接并且成功提交到 115 离线下载，就拦截 MoviePilot 原下载。搜索不到、匹配不达标、115 提交失败时，不做任何拦截，MoviePilot 会继续走原来的 PT/qBittorrent 流程。

当前版本：`1.0.9`

## 文件结构

```text
plugins/mp115cloud/__init__.py
plugins.v2/mp115cloud/__init__.py
package.json
package.v2.json
README.md
```

## 在线安装

推荐把本目录作为一个 GitHub 插件仓库发布，仓库根目录保持下面结构：

```text
plugins/mp115cloud/__init__.py
plugins.v2/mp115cloud/__init__.py
package.json
package.v2.json
README.md
```

发布到 GitHub `main` 分支后，在 MoviePilot V2 的插件市场设置里添加你的仓库地址，例如：

```text
https://github.com/duyao9992/MP115Cloud-MoviePilot-Plugins
```

仓库地址不要带 `.git`，也不要填 `raw.githubusercontent.com` 链接。

刷新插件市场后搜索 `115 云下载接管` 或 `MP115Cloud`，再从界面安装。这样后续升级只需要提交新版代码并同步 `package.v2.json` 和插件里的 `plugin_version`。

## 手动安装

1. 把 `plugins.v2/mp115cloud` 复制到你的 MoviePilot 插件仓库或容器插件目录。
2. 如果你维护自己的插件市场，把 `package.v2.json` 中的 `MP115Cloud` 合并到插件仓库根目录的 `package.v2.json`。
3. 重启 MoviePilot，或在插件页面重载插件。

## 推荐配置

插件页面里最关键的字段：

```text
启用插件: 打开
仅处理订阅: 打开
成功后拦截原下载: 打开
成功后更新订阅状态: 打开
搜索页/首页 URL: http://your-internal-site/ 或 http://your-internal-site/search?q={keyword}
搜索关键词模板: {title}
必须包含中文字幕: 打开
结果列表路径: data
JSON 磁链字段路径: magnet
JSON 标题字段路径: title
JSON 做种数字段路径: seeders
115 Cookie: UID=...; CID=...; SEID=...;
115 目标目录 ID: 可空，或填你希望保存的 115 文件夹 cid
```

如果页面搜索必须像浏览器一样在顶部搜索框输入关键词，就把 `搜索页/首页 URL` 填成首页地址，不要强行拼 `/search?q=`。插件会先打开这个页面，识别 `<form>` 里的搜索输入框，再按页面声明的 `action` 和 `method` 提交搜索。

`搜索页/首页 URL` 支持占位符：

```text
{keyword}       URL 编码后的搜索词
{raw_keyword}   原始搜索词
{title}         媒体标题
{torrent_title} MoviePilot 已选中的 PT 资源标题
{year}          年份
{season}        季
{episode}       集
{tmdbid}
{imdbid}
{doubanid}
```

如果你的搜索页面是普通 HTML，插件会自动解析：

```html
<a href="magnet:?xt=urn:btih:..." title="Example.Movie.2026.1080p">
  Example.Movie.2026.1080p
</a>
```

也支持把磁链放在 `data-clipboard-text`、`data-url` 等属性里。标题优先取 `title`、`aria-label`、`data-title`、`data-name`，其次取链接文本。

从 `0.2.0` 起，如果搜索结果页没有直接放磁链，只放了详情页链接，插件会：

1. 解析搜索页里的结果链接。
2. 按标题相似度、4K/杜比视界/字幕等优先级排序。
3. 进入前几个详情页。
4. 在详情页继续查找 `magnet:`，包括 `href`、`data-clipboard-text`、`data-url` 等属性。

从 `0.3.1` 起，插件按这个站点的实际路径处理：

1. 用 MoviePilot 媒体标题搜索影片，例如 `侵略机器`，不是用 PT 种子完整标题。
2. 首页没有标准 `<form>` 时，优先按真实入口访问 `/s/影片名/`，例如 `/s/侵略机器/`。
3. 从搜索结果进入影片详情页。
4. 在影片详情页的“磁力”资源列表里按 `中文字幕/中字/简繁`、`4K/2160p`、`杜比视界/DV/HDR` 优先筛选资源条目。
5. 进入被选中的资源磁力页。
6. 从最终页面的 `复制磁力` 按钮、`data-clipboard-text`、`href` 或页面文本里提取真实 `magnet:`。

从 `0.3.2` 起，影片详情页和资源磁力页请求会沿用前面搜索页返回的 Cookie，并把上一页 URL 作为 `Referer` 带上；如果站点临时重置连接，会自动重试一次。

从 `0.3.3` 起，搜索站点 Cookie 会作为 cookie jar 传递，不再固定塞进 `Cookie:` 请求头。这样搜索页返回的新 Cookie 会继续合并到后续影片详情页和资源页请求里。

从 `0.3.4` 起，影片详情页只保留“磁力”标签里的资源入口，自动排除 `百度/夸克/迅雷/UC/阿里` 等网盘入口。插件会先点击资源列表里的磁力条目，再进入最终页面提取“复制磁力”对应的真实链接；最终页面里的 URL 编码、JS 转义、base64 或 hash 字段也会尝试解析。

从 `0.3.5` 起，如果首页/搜索框入口被站点临时重置连接，插件会继续尝试 `/s/影片名/` 等真实搜索入口，不会因为首页失败就直接回落 MoviePilot 原流程。普通请求也会最多重试 3 次。

从 `0.3.6` 起，匹配影片结果和资源条目时会同时参考 MoviePilot 媒体标题、英文原名、别名和年份。比如 `复仇者联盟3：无限战争`、`复仇者联盟：无限之战`、`Avengers: Infinity War` 会被作为同一组目标名参与匹配；如果候选标题年份不同，会自动降权。

从 `0.3.7` 起，可以在插件页面填写 `搜索站点代理 URL`，只让磁力站搜索、影片详情页和资源磁力页请求走代理，不影响 115 离线提交和 MoviePilot 原下载流程。示例：`http://192.168.31.18:7890`、`socks5h://192.168.31.18:7890`。注意 Docker 容器内的 `127.0.0.1` 是容器自身，如果代理在 NAS 或路由器上，通常需要填写局域网 IP。

从 `0.3.8` 起，资源条目增加防串片保护：不能只靠 `4K/DV/HDR/中字` 等质量关键词通过。如果资源标题和已进入的影片页标题都与订阅目标差异过大，插件会跳过这些资源并回落 MoviePilot 原流程，避免 `康斯坦丁` 被错误接管成其它影片。

从 `0.3.9` 起，影片页候选也会做更严格的标题证据判断：低相似度且没有中文片名、英文片名或别名交集的候选会直接跳过。这样搜索入口临时失败、泛化页面返回 `挽救计划`、`洛杉矶劫案` 等无关影片时，不会继续进入资源页并误提交 115。

从 `0.4.0` 起，系列片会做通用部数识别和年份硬校验：同一中文基底下的 `1/2/3`、`第一部/第二部/第三部` 等明确部数不能互相串片；目标第一部可接受无编号写法，但会拒绝候选明确写成第二部、第三部的资源。磁力资源默认还必须识别到中文字幕，并按 `4K+杜比视界 > 4K > 1080P` 选择。

从 `0.4.1` 起，SeedHub 这类 `/s/片名/` 真实搜索结果页一旦命中详情链接或磁链，会立即作为搜索结果使用，不再被后续 `?keyword=`、`?q=` 等可能重定向到首页的泛化入口覆盖。

从 `0.4.2` 起，如果 115 返回 `任务已存在，请勿输入重复的链接地址`，插件会视为 115 已有该离线任务并继续接管成功，不再回落 MoviePilot。正确影片页已解析到资源磁链后也会停止进入后续影片页候选，避免相近片名页面拖慢流程。

从 `0.4.3` 起，`/s/片名/` 真实搜索入口遇到连接重置会增加重试；如果 `?wd=`、`?keyword=` 等入口返回的是不含搜索词的首页泛化页，插件不会再把它当作有效搜索结果。

从 `0.4.4` 起，`?wd=`、`?keyword=` 等 query 搜索入口即使页面正文包含关键词，也必须解析出匹配的影片候选才会被接受；插件初始化日志和 `/health` 会显示当前版本，方便确认容器实际加载的版本。

从 `0.4.5` 起，精确完整标题路径没有可用候选时会自动降级搜索通用系列关键词，例如 `封神Ⅰ：朝歌风云 -> 封神第一部 -> 封神`、`封神Ⅱ：战火西岐 -> 封神第二部 -> 封神`；降级只改变搜索词，最终仍按原影片标题、年份和系列部数严格筛选，避免第一部/第二部串片。

从 `0.4.6` 起，降级搜索只在精确搜索没有命中匹配影片/资源入口时触发；如果已经命中正确影片页但资源磁力页被站点断开连接，不会再降级搜索系列基底关键词。资源页拿到第一个有效磁链后会立即停止请求后续资源页，减少 `Connection reset by peer`。

从 `0.4.7` 起，资源页请求如果连续被 `Connection reset by peer` 断开，插件会在 `requests` 重试后自动改用 `curl` 兜底请求一次，提升 SeedHub `link_start` 磁力页取链成功率。

从 `0.4.8` 起，资源磁力页第一次 `requests` 失败后会立即尝试 `curl` 兜底，成功就不再继续等待后续重试；资源页 `requests` 重试从 6 次降为 3 次，并保留 `curl` 兜底。该改动只减少网络失败等待，不改变标题、年份、字幕和清晰度筛选规则。

从 `0.4.9` 起，电视剧只接管目标季整季/全集包：接受 `S01`、`第一季[全08集]`、`S01E01-08` 这类整季资源，拒绝 `S01E01`、`第01集`、`EP01` 等单集资源，以及非目标季或多季合集。被拒绝后会返回空候选，让 MoviePilot 原流程处理单集追更。

从 `0.4.10` 起，电视剧会从 MoviePilot 订阅记录恢复完整订阅标题和目标季，例如 `黑袍纠察队 S02` 会用 `黑袍纠察队 S02` 搜索，并把目标季识别为第二季；标题匹配仍使用基础剧名，季数单独硬过滤。如果电视剧没有识别到目标季，插件不会接管。影片详情页也会先按目标季过滤，避免第二季订阅进入第五季详情页；`第01-02集`、`S05E01-02` 等部分集范围不再算整季包。

从 `0.4.11` 起，修复电视剧 `S02/S03` 精确搜索空页误判：`/s/{keyword}/` 路径里的空格会按 `%20` 编码，不再写成 `+`；电视剧搜索页只有基础剧名或其它季时，不再算“精确命中”；如果目标季精确页没有资源，会继续尝试 `第二季/第三季`、无空格 `S02/S03`、基础剧名等关键词，但最终仍按目标季硬过滤。

从 `0.4.12` 起，电视剧整季包识别按通用季数/集数范围判断：支持 `S02/S03/S20` 等两位季数，也支持 `EP01-08`、`EP01-20`、`S20E01-20`、`第01-20集` 这类从第 1 集开始且跨度足够的整季范围；继续拒绝 `EP03-10`、`EP01-02` 等部分包。插件还会记录同一订阅目标的近期成功提交，避免 MoviePilot 连续触发时对同一季重复提交到 115。

从 `0.4.13` 起，增强电视剧重复触发保护：同一订阅目标某一季成功提交后，会同时记录“带季数”和“不带季数”的近期提交键。MoviePilot 紧接着再次触发同一剧名但缺少季数的事件时，插件会直接拦截跳过，避免已经成功提交 115 后又回落原流程；带明确 `S02/S03/S20` 的其它季仍按具体季数独立处理。

从 `0.4.14` 起，电视剧事件如果缺少目标季，会在打印“开始搜索”和发起任何资源站请求之前直接回落 MoviePilot 原流程，避免日志看起来像“成功提交后又搜索了一次”。如果这是刚刚成功提交过的同一订阅目标，则仍由 `0.4.13` 的近期提交保护直接拦截重复事件。

从 `1.0.0` 起，作为稳定版发布；本版本仅升级版本号，不改变现有匹配、搜索、提交和拦截规则。

从 `1.0.1` 起，如果 `/s/{keyword}/` 直接返回 `HTTP 403` 或其它请求失败，插件不会立刻回落 MoviePilot；会先尝试同关键词备用入口（如 `?wd=`、`?keyword=`、`/search`），再尝试通用降级关键词，并继续使用原有标题、年份、目标季、字幕和清晰度过滤。

从 `1.0.2` 起，电视剧资源年份不再作为硬拒绝条件。电视剧资源标题里的年份常常是目标季上映年份，而 MoviePilot 媒体年份可能是整部剧首播年份；电视剧会以目标季为硬条件继续筛选，电影仍保留年份冲突硬拒绝。

从 `1.0.3` 起，同一订阅同一季整季包提交成功后，会按“订阅 ID + 季号”记录近期提交，不再依赖中文名、英文名、资源标题或年份；后续同季单集或资源标题事件会直接拦截，避免重复提交或回落 MoviePilot。

从 `1.0.7` 起，兼容缺少 `app.helper.subscribe` 的 MoviePilot 版本；插件安装后不会再因为 `SubscribeHelper` 导入失败而无法显示在“已安装”。搜索、匹配、提交和拦截规则不变。

从 `1.0.8` 起，同一订阅目标短时间内被 MoviePilot 连续触发多个候选事件时，插件仍会逐个拦截重复下载，但 60 秒内只打印一次“近期已提交”日志，避免日志刷屏。

从 `1.0.9` 起，演练模式日志会明确显示“未提交 115”，不再写成“已提交 115”；重复拦截日志也改为“订阅目标已由 115 接管”，避免误解为后续每个候选资源都已经提交过。

相关配置：

```text
进入详情页抓磁链: 打开
最多详情页数: 5
```

日志会输出：

```text
[MP115Cloud] 搜索页解析: 直接磁链 0 条，详情链接 20 条
[MP115Cloud] 详情页候选 1: score=...
[MP115Cloud] 磁力资源链接筛选: 80 / 285
[MP115Cloud] 资源页磁链解析: 1 条
```

如果这个测试网站需要登录，把 Cookie 或 Authorization 放进 `搜索页请求头`：

```text
Cookie: session=xxx; token=yyy
User-Agent: Mozilla/5.0
```

也可以直接填 `搜索站点 Cookie` 字段。这个字段只用于搜索站点；`115 Cookie` 只用于提交 115 离线任务，两个 Cookie 不要混在一个框里。

如果 NAS 或 MoviePilot 容器解析到的 IP 和电脑浏览器实际访问的 IP 不一致，可以填 `DNS/IP 覆盖`，每行一个：

```text
www.example.test=192.168.31.10
```

插件会用 `curl --resolve host:port:ip` 请求搜索页，保留原始 Host/SNI，不再依赖容器 DNS 对这个域名的解析。

从 `0.2.3` 起，同一行可以填多个候选 IP，插件会按顺序尝试：

```text
www.example.test=192.168.31.10,192.168.31.11,192.168.31.12
```

从 `0.3.1` 起，如果首页没有标准 `<form>`，插件会优先尝试 SeedHub 这类站点常见的路径搜索入口：

```text
/s/影片名/
```

如果这个入口没有命中，再自动试几个常见搜索入口，例如：

```text
/?keyword=...
/?q=...
/search?q=...
/search?keyword=...
```

同时会过滤豆瓣等外站链接、分页数字和导航短标题，避免把它们误当成资源详情页。

插件即使不填请求头，也会默认带上常见浏览器 `User-Agent`、`Accept`、`Accept-Language` 和 `Referer`。如果搜索请求成功返回，日志会出现类似：

```text
[MP115Cloud] 搜索页已返回: HTTP 200, 内容 12345 字符, 类型 text/html, URL ...
```

如果日志仍然是 `搜索请求失败: ... timed out`，说明 MoviePilot 容器还没有拿到页面，需要先处理容器网络、DNS 或代理；这时还没有进入 HTML/JSON 解析阶段。

从 `0.1.2` 起，搜索前还会记录容器内的域名解析结果：

```text
[MP115Cloud] 搜索请求准备: URL=..., host=..., port=443, DNS=['IPv4 x.x.x.x', 'IPv6 xxxx::...']
```

如果电脑、手机和 NAS 都在同一网络但只有 NAS 超时，优先对比这里的 DNS/IP 与电脑浏览器实际访问的 IP，通常能定位到 hosts、DNS、IPv6 路由或容器网络差异。

如果它碰巧返回 JSON，也继续支持：

```json
{
  "data": [
    {
      "title": "Example.Movie.2026.1080p",
      "magnet": "magnet:?xt=urn:btih:...",
      "seeders": 12
    }
  ]
}
```

就使用默认路径：

```text
结果列表路径: data
JSON 磁链字段路径: magnet
JSON 标题字段路径: title
JSON 做种数字段路径: seeders
```

## 选择规则

当一个搜索页里有多个磁链时，插件会先判断标题、年份和系列部数；通过后，资源标题必须满足中文字幕和清晰度规则。默认选择顺序：

```text
4K + 杜比视界 / Dolby Vision / DV
4K / 2160p / UHD
1080P / FHD
中文字幕 / 中字 / 简中 / 繁中 / 简繁 / CHS / CHT / Chinese
BluRay / WEB-DL / H.265 / x265 / HEVC
```

默认排除：

```text
枪版 / 抢先 / TC / TS / CAM / HDTC / HDCAM / 清晰版 / 机翻
```

这些都能在插件配置页修改，格式是：

```text
关键词:分数
```

## 流程说明

1. MoviePilot 订阅先按原来的 PT 站规则搜索、过滤并选出候选资源。
2. 插件监听 `ChainEventType.ResourceDownload`，只在真正添加下载器前触发。
3. 插件用媒体标题请求你配置的非公开搜索页。
4. 插件进入影片页和资源磁力页，确认中文字幕后按 4K+杜比视界、4K、1080P 顺序选择磁链并提交到 115 离线任务。
5. 115 返回成功后，插件设置 `event_data.cancel = True`，MoviePilot 不再添加 qBittorrent 下载。
6. 任一步失败，插件不设置 cancel，MoviePilot 继续原流程。

## 测试接口

MoviePilot 会给插件 API 加上插件前缀，常用测试：

```text
GET  /api/v1/plugin/MP115Cloud/health
GET  /api/v1/plugin/MP115Cloud/search?keyword=Example.Movie.2026
POST /api/v1/plugin/MP115Cloud/submit?magnet=magnet:?xt=urn:btih:...
```

具体前缀以你的 MoviePilot 部署为准。

## 注意事项

- 115 Cookie 至少需要包含 `UID`、`CID`、`SEID`。
- `演练模式` 会完成搜索和匹配，但不会真正提交 115；要真实复制磁力到 115 必须关闭。
- `成功后更新订阅状态` 默认开启。电影会直接完成订阅；剧集会按本次事件的集数更新订阅 note 和缺失集数。
- `必须包含中文字幕` 默认开启。资源标题无法识别中文字幕或清晰度时会回落 MoviePilot 原流程。
- 如果 HTML 页面里的磁链没有标题或链接文本，默认不会接受无标题结果，避免错转。确实可信时再打开 `允许无标题结果`。
